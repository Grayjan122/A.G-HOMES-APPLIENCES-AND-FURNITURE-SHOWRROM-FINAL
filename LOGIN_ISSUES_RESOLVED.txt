🔄 HOW IT WORKS NOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LOGIN FLOW (Updated):

1. User enters username and password
   ↓
2. Frontend sends login request
   ↓
3. Backend finds user by username
   ↓
4. Backend retrieves stored password (encrypted)
   ↓
5. Backend tries multiple decryption methods:
   • Plain text match?
   • MD5 match?
   • SHA256 match?
   • password_hash match?
   ↓
6. If password matches:
   • Update active_status to "Online" ✅
   • Return user data (without password)
   • Log success
   ↓
7. Frontend receives user data
   ↓
8. Frontend stores session data
   ↓
9. Frontend attempts to call OnlineState (backup)
   ↓
10. Frontend creates audit log
    ↓
11. User redirected to dashboard
