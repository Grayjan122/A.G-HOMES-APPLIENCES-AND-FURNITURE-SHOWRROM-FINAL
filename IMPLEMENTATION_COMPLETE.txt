🎯 HOW IT WORKS (SUMMARY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: User logs in
  → InactivityLogout component activates
  → Starts tracking user activity
  → Sets 1-hour timer

Step 2: User is active (normal usage)
  → Every mouse move, click, keypress, scroll, or touch
  → Resets the 1-hour timer
  → User can work indefinitely

Step 3: User becomes inactive (59 minutes)
  → Warning modal appears
  → Shows countdown: 60 seconds
  → Two options: "Stay Logged In" or "Logout Now"

Step 4a: User clicks "Stay Logged In"
  → Modal closes
  → Timer resets to 1 hour
  → User continues working

Step 4b: User does nothing (countdown reaches 0)
  → Auto logout triggered
  → User status set to "Offline"
  → Audit log entry created
  → Redirected to login page

Step 5: Browser close
  → User closes browser or tab
  → beforeunload event triggers
  → User status set to "Offline"
  → Session cleaned up
