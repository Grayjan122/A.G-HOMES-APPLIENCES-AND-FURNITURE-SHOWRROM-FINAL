Mobile Browsers:
  1. pagehide ✅ (most reliable on mobile)
  2. beforeunload ⚠️ (less reliable)

Desktop Browsers:
  1. beforeunload ✅ (reliable)
  2. pagehide ✅ (also works)

Back Button:
  1. popstate ✅ (all browsers)


┌─────────────────────────────────────────────────────────────────────┐
│                   SENDBEACON ADVANTAGES                             │
└─────────────────────────────────────────────────────────────────────┘

Regular AJAX vs sendBeacon:

❌ Regular AJAX (fetch/axios):
   - Can be cancelled during page unload
   - May not complete if page closes fast
   - Can block page unload
   - Not guaranteed delivery

✅ sendBeacon:
   - Specifically designed for unload
   - Guaranteed to send (non-blocking)
   - Queued by browser
   - Completes even after page closes
   - Perfect for logout scenarios


┌─────────────────────────────────────────────────────────────────────┐
│                     SESSION CLEANUP                                 │
└─────────────────────────────────────────────────────────────────────┘

Every logout scenario executes:

  sessionStorage.clear()
       │
       ├─→ Removes: user_id
       ├─→ Removes: baseURL
       ├─→ Removes: all session data
       │
       └─→ Result: User must login again


┌─────────────────────────────────────────────────────────────────────┐
│                  SECURITY PROTECTION FLOW                           │
└─────────────────────────────────────────────────────────────────────┘

Back Button Protection:

  Page Load
       │
       ▼
  history.pushState(null, '', location)
       │
       ▼
  User clicks back (←)
       │
       ▼
  popstate event detected
       │
       ├─→ Send logout beacon
       ├─→ Clear session
       ├─→ Push state again (prevent back)
       └─→ Redirect to login
       
  Result: Cannot navigate back to authenticated page!
