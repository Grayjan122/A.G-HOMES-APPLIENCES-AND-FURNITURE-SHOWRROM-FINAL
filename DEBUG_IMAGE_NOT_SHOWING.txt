SCENARIO A: Image URL is Incomplete in Step 3
----------------------------------------------
Problem: Upload Response shows incomplete URL
Example: "path": "https://res.cloudinary.com/dqxygi0yz/image/upload/"

CAUSE: Cloudinary upload failed or returned bad data

FIXES:
  1. Check .env.local has correct API Secret
  2. Verify internet connection
  3. Check Cloudinary dashboard for errors
  4. Try a different image file


SCENARIO B: Image URL is Complete in Steps 3-6, but Incomplete in Step 9
-------------------------------------------------------------------------
Problem: URL gets truncated between saving and fetching

CAUSE: Database column is too short (VARCHAR(255))

FIX: Update database column:
  ALTER TABLE products 
  MODIFY COLUMN product_preview_image VARCHAR(500) OR TEXT;


SCENARIO C: All URLs are Complete, But Image Still Doesn't Show
----------------------------------------------------------------
Problem: URLs are correct everywhere but image not visible

POSSIBLE CAUSES:

1. CORS Issue
   └─ Open image URL directly in new tab
   └─ If it doesn't load, Cloudinary has access restriction
   └─ Check Cloudinary dashboard → Security settings

2. CSS/Display Issue
   └─ Right-click on image placeholder
   └─ Select "Inspect Element"
   └─ Check if <img> tag has correct src
   └─ Check CSS for "display: none" or "visibility: hidden"

3. React State Issue
   └─ productList might not be updating
   └─ Check if other product data shows correctly

4. Cache Issue
   └─ Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
   └─ Try in Incognito/Private browsing mode


SCENARIO D: Image Shows as Broken Image Icon
---------------------------------------------
Problem: <img> tag exists but shows broken image

CAUSE: Image URL is malformed or inaccessible

CHECKS:
  1. Copy image URL from browser inspector
  2. Paste in new tab - does it load?
  3. Check for typos in URL
  4. Verify image wasn't deleted from Cloudinary


SCENARIO E: getImageUrl Returns null
-------------------------------------
Problem: Step 10 shows "null"

CAUSE: getImageUrl() helper function rejecting the URL

CHECKS:
  1. URL is exactly "Nothing as for now" (rejected)
  2. URL is null or undefined (rejected)
  3. Check getImageUrl() logic in code

FIX 1: Test Image URL Directly
-------------------------------
1. Look at console log #9: "First product image URL: [URL]"
2. Copy that URL
3. Paste it in a new browser tab
4. Does the image load?

   YES → Image is accessible, problem is in display code
   NO → Image URL is wrong or Cloudinary has restrictions


FIX 2: Check Browser Network Tab
---------------------------------
1. Open F12 → Network tab
2. Refresh page
3. Look for failed image requests (red)
4. Click on failed request
5. Check error message


FIX 3: Inspect Image Element
-----------------------------
1. Right-click on where image should be
2. Select "Inspect" or "Inspect Element"
3. Look for <img> tag
4. Check src attribute value
5. Is it the correct Cloudinary URL?


FIX 4: Hard Refresh
-------------------
1. Press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
2. This clears browser cache
3. Check if image appears now


FIX 5: Check Cloudinary Dashboard
----------------------------------
1. Go to: https://cloudinary.com/console
2. Click "Media Library"
3. Navigate to: ag-homes/products
4. Is your image there?
5. Click on image → Get URL
6. Compare with database URL

Issue #1: Database Column Too Short (75% of cases)
---------------------------------------------------
Symptoms:
  - URL complete during upload
  - URL incomplete when fetched from database
  - URL ends with "/upload/"

Solution:
  ALTER TABLE products 
  MODIFY COLUMN product_preview_image TEXT;


Issue #2: Cloudinary URL Not Public (15% of cases)
---------------------------------------------------
Symptoms:
  - URL looks correct
  - Opening URL in new tab shows "Access Denied"

Solution:
  1. Go to Cloudinary dashboard
  2. Settings → Security
  3. Check "Allow public access"


Issue #3: React Not Re-rendering (5% of cases)
-----------------------------------------------
Symptoms:
  - URL is correct
  - Image shows after page refresh
  - Not showing after upload

Solution:
  - Already implemented (GetProduct() called after save)
  - Try hard refresh


Issue #4: CORS Policy (3% of cases)
------------------------------------
Symptoms:
  - Console shows CORS error
  - Image blocked by browser

Solution:
  - Check Cloudinary CORS settings
  - Verify domain is allowed


Issue #5: Cache Issue (2% of cases)
------------------------------------
Symptoms:
  - Everything looks correct
  - Image just won't show

Solution:
  - Hard refresh: Ctrl+Shift+R
  - Clear browser cache
  - Try incognito mode
